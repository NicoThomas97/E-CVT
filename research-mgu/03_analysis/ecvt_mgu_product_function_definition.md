# e-CVT MGU 整车系统功能定义

最后更新：2026-07-16  
定位阶段：产品功能定义 / 预研输入  
证据基础：`url_key_info_integrated_dedup_summary.md`、`technical_architecture.md`、`ecvt_mgu_validation_plan.md`、`ecvt_mgu_visual_storyboard.md`

## 1. 产品定义

本产品定义为一套面向整车厂的 **智能脚感一致性 e-CVT MGU 平台**，不是单一电机，也不是单一变速箱。系统由中置电机、电子控制连续变速机构、完整传感器包、控制算法、骑手 HMI、App/设置页、OEM 标定工具和诊断能力组成。

核心产品叙事：

> 以踏频管理为基础，以脚感一致性为目标，结合多传感器识别骑行者意图，自动调整助力和传动比，让骑手在巡航、冲刺、爬坡等不同场景中获得自然、连续、可控的踩踏体感。

产品对标 Avinox MG Concept、Gobao X1P、Owuru Ride、Pinion MGU、Bosch eShift 等能力集合，但目标不是复制某一个竞品，而是把“电机集成 e-CVT”的整车系统能力定义完整。

## 2. 产品原则

| 原则 | 定义 | 产品含义 |
| --- | --- | --- |
| 少控制，强自动 | 骑手不需要频繁选择档位 | 默认由系统处理起步、巡航、爬坡、滑行恢复和急加速 |
| 踏频管理为基础 | 目标 rpm 是重要控制入口 | 巡航和轻负载场景优先维持骑手设定或模式默认踏频 |
| 脚感一致性为目标 | 不把固定踏频当作所有场景的唯一目标 | 爬坡和重载场景优先让踩踏力矩处于不过重、不过空的舒适中间区 |
| 场景化控制优先级 | 巡航看踏频，冲刺看意图，爬坡看脚感 | 根据骑手输入和路况切换目标踏频、目标力矩区间、传动比和电机转矩策略 |
| 高级可调 | 普通骑行简单，深度设置可进 App/设置页 | 骑手可调目标 rpm、模式性格、响应速度、起步偏好等 |
| 虚拟档隐藏 | 不把虚拟档作为默认主界面 | 虚拟齿比/虚拟档位用于算法建模、工程标定和必要时的高级模式 |
| 平台化 | 先定义总功能池，不先细分车型版本 | 后续可按 eMTB、城市、旅行、货运等整车场景标定 |

## 3. 功能总池

### 3.1 骑手控制功能

骑行中主控项：

- 骑行模式：至少包括舒适和运动两个方向，后续可扩展为标准、自动、节能、爬坡等。
- 目标踏频 rpm：支持精确 rpm 设置，初始参考范围为 40-120 rpm，步进参考 5 rpm。
- 助力等级：与传动策略联动，而不是只改变电机输出功率。
- 临时偏好调整：允许骑手在当前骑行状态下请求更轻、更重、更运动或恢复默认。

App/设置页高级项：

- 目标踏频范围和默认值。
- 舒适/运动模式下的目标 rpm、响应速度、自动变速积极程度。
- 起步速比偏好。
- 滑行恢复策略偏好。
- 冲刺/急加速响应偏好。
- 自动策略被手动干预后的保持时间和学习程度。

不作为默认骑行主界面的功能：

- 固定显示 4/8/12/13 档虚拟档位。
- 让骑手实时管理真实传动比。
- 把 Pre.Select、Start.Select、Roll-Shift 等功能拆成一堆需要用户理解的独立模式。

### 3.2 自动变速与传动逻辑

系统基础逻辑不是单一恒踏频闭环，而是“场景化脚感控制”：系统先判断当前处于巡航、冲刺、爬坡、起步或滑行恢复等场景，再决定目标踏频、目标踩踏力矩区间、e-CVT 传动比和电机助力的优先级。

必须具备的传动场景：

| 场景 | 系统行为 | 体感目标 |
| --- | --- | --- |
| 平路巡航 | 维持目标 rpm，减少频繁变比 | 稳、安静、省心 |
| 轻微起伏 | 小幅提前调整传动比 | 不让骑手明显感到坡度变化 |
| 爬坡 | 根据坡度、力矩和速度变化动态调整目标踏频和传动比 | 让踩踏力矩保持在舒适中间区，不沉、不空 |
| 急加速/冲刺 | 识别快速加力和速度请求，允许真实踏频短时上扬，电机快速响应，齿比平滑跟随 | 像汽车自动变速箱降档一样尊重骑手加速意图 |
| 冲刺后恢复巡航 | 骑手力矩下降后目标踏频回到巡航值，保持较高齿比 | 以更高车速、更低踏频进入高速巡航 |
| 停车起步 | 自动进入舒适起步传动比 | 起步不重、不空、不突兀 |
| 滑行恢复 | 停止踩踏或下坡滑行时，按车速预选恢复踩踏所需传动比 | 一踩上去就是合适阻力 |
| 高负载换比 | 高扭矩、高助力、爬坡时仍可连续变比 | 不断力、不顿挫、不触发不必要保护 |
| 曲柄相位窗口 | 根据曲柄位置选择更自然的变比时机 | 减少脚感突兀和踩踏冲击 |

场景化控制优先级：

- 平路巡航：目标踏频优先。系统以骑手设定或模式默认 rpm 为主目标，减少齿比和助力扰动，让骑行稳定、省心、安静。
- 平路突然加速：骑手意图优先。骑手踩踏力矩快速上升时，系统不急于把真实踏频压回原巡航值，而是允许踏频短时上扬；电机转矩先快速响应，e-CVT 齿比再平滑增加。
- 冲刺结束：恢复巡航优先。骑手踩踏力矩下降后，目标踏频回到巡航值；由于齿比已经升高，车辆可在更高速度下以舒适踏频巡航。
- 陡坡爬坡：脚下力矩优先。系统不死守固定踏频，而是通过动态目标踏频、齿比和电机转矩协同，把骑手踩踏力矩维持在舒适中间区。
- 换比窗口：曲柄相位优先。变比动作尽量放在踩踏力矩低谷 / 曲柄 12 点附近，降低脚感冲击。

### 3.3 骑行意图识别

意图识别是本系统区别于普通自动换挡的重点研究方向。系统不应只根据速度或踏频机械地变比，而要综合判断骑手想做什么。

核心输入：

- 踏频。
- 踩踏力矩。
- 曲柄位置/相位。
- 轮速。
- 坡度/IMU。
- 刹车状态。
- 电机电流。
- 电机转速。
- 系统温度。
- 电池状态。

首批应识别的意图：

| 骑行意图 | 典型信号 | 控制策略 |
| --- | --- | --- |
| 舒适巡航 | 踏频稳定、力矩低到中、速度稳定 | 保持目标 rpm，降低变比和助力扰动 |
| 运动巡航 | 目标 rpm 偏高、力矩中高、速度持续提升 | 维持较高目标 rpm，助力响应更积极 |
| 急加速/冲刺 | 力矩快速上升、踏频变化快、速度请求明显 | 允许真实踏频上扬，电机快速响应，齿比平滑增加 |
| 冲刺后恢复巡航 | 力矩下降、车速已提高、踏频需要回落 | 目标踏频回到巡航值，保持较高齿比进入高速巡航 |
| 爬坡 | 坡度上升或保持高坡度、速度下降、力矩持续偏高 | 优先控制踩踏力矩落入舒适区，目标踏频可动态变化 |
| 技术路段 | 力矩波动、速度低、IMU/刹车变化频繁 | 降低误动作，保持可预测和稳定支撑 |
| 滑行恢复 | 无踩踏、速度存在、即将恢复踩踏 | 预选合适传动比 |
| 起步 | 低速或静止、踏频从零开始 | 进入舒适起步传动比 |

个人判断纳入定义：Auto.Shift.Pro 类方案的问题在于不能充分自动识别骑行者意图。本系统应通过多传感器把“骑手意图识别”做成底层控制能力，而不是要求骑手频繁手动告诉系统。

### 3.4 骑行体感目标

本产品的体验评价不应只看变速速度，而应围绕以下主观体感：

- 自然：用户感觉是腿变强，而不是机器在抢控制权。
- 平顺：传动比和助力变化不产生明显顿挫、拖拽、空踩或突兀电感。
- 稳定：巡航时目标踏频稳定，系统不过度频繁调整。
- 运动：冲刺、急加速时响应足够快，允许踏频上扬并逐步升齿比，让骑手感到车是被自己踩起来的。
- 脚感一致：陡坡和重载时，系统优先把踩踏力矩维持在舒适中间区，而不是机械维持固定踏频。
- 可控：系统误判时，用户可以通过 rpm、模式或临时偏好快速纠正。
- 安静：高负载和高踏频下噪音可控，45 dB 级别可作为后续研究目标之一，但不能在未验证前作为量产承诺。
- 可信：传感器异常、低电、过热或系统降级时，骑手仍知道车辆会怎样响应。

### 3.5 整车系统功能

系统应作为整车平台能力交付给 OEM，而不只是一个驱动单元：

- 电机与 e-CVT 传动集成。
- 支持单速链条或皮带，取消传统后拨和多片飞轮。
- 支持防盗锁止，目标是断电状态下仍能保持锁止能力。
- 支持推行辅助和倒车辅助。
- 支持可控再生制动/能量回收，但作为增强功能，不作为第一主卖点。
- 支持显示屏、遥控器、App、BMS、VCU/整车控制器联动。
- 支持诊断、故障记录、传感器异常识别和安全降级。
- 支持整车厂标定不同车感包。

### 3.6 OEM / 工程标定功能

OEM 和工程端需要能定义整车“性格”，而不是只能接受一个固定控制逻辑。

必须支持的标定项：

- 目标踏频默认值和可调范围。
- 踩踏力矩舒适区目标。
- 舒适/运动模式下的目标 rpm。
- 助力映射。
- 传动比变化速度。
- 自动变速积极程度。
- 起步速比默认值。
- 滑行恢复策略。
- 爬坡策略。
- 冲刺策略。
- 冲刺后恢复巡航策略。
- 目标踏频动态变化策略。
- 手动临时覆盖后的保持时间。
- 用户偏好学习强度。
- 传感器阈值和故障降级策略。

起步速比应通过大量实验确定基础默认值，也允许整车品牌自定义，或允许高级用户在 App/设置页中进行个性化选择。

## 4. 控制接口定义

### 4.1 骑行中 HMI

骑行中界面应保持简单，避免把复杂策略暴露给用户。

默认显示/控制：

- 当前模式。
- 当前目标踏频 rpm。
- 助力等级。
- 临时偏好调整。
- 系统状态提示。

不建议骑行中常驻显示：

- 真实传动比。
- 虚拟档位编号。
- 复杂策略名称。
- 工程参数。

### 4.2 App / 设置页

App/设置页承载高级骑手控制：

- 目标踏频范围。
- 舒适/运动模式默认 rpm。
- 自动变速响应速度。
- 起步偏好。
- 滑行恢复偏好。
- 冲刺响应偏好。
- 是否允许系统学习用户习惯。
- 诊断信息和固件状态。

### 4.3 OEM 标定工具

OEM 标定工具面向整车品牌和工程调试：

- 控制策略配置。
- 默认值配置。
- 传感器阈值配置。
- 车感包配置。
- 故障码读取。
- 数据日志导出。
- 台架/样车测试参数记录。

### 4.4 控制输出

系统输出应覆盖：

- 电机助力。
- 电机转速/扭矩控制。
- e-CVT 目标传动比。
- e-CVT 实际传动比反馈。
- 再生制动力。
- 锁止状态。
- 推行/倒车状态。
- 故障降级状态。

## 5. 安全与降级

系统功能越自动，越需要明确边界。

基础要求：

- 手动临时覆盖优先级高于系统自动判断。
- 刹车状态优先级高于助力和再生制动请求。
- 传感器异常时进入可骑行安全模式。
- 低电量时保留基本传动能力，避免车辆卡在不可骑状态。
- 过热时优先降低助力和变比响应速度，而不是突然中断。
- 再生制动在过弯、湿滑、低附着、低速状态下必须限制介入。
- 防盗锁止、倒车辅助、推行辅助必须有清晰的状态提示和误触发保护。

推荐降级策略：

| 故障 | 降级行为 |
| --- | --- |
| IMU 异常 | 停用坡度/姿态增强，保留踏频、力矩、轮速基础自动控制 |
| 力矩传感器异常 | 降低助力，进入保守固定目标传动比或低风险自动模式 |
| 曲柄位置异常 | 停用曲柄相位最优换比，保留连续变比但降低响应速度 |
| 轮速异常 | 限制自动变速和再生制动，提示维修 |
| 温度过高 | 降低助力和变比响应，记录热事件 |
| 电池低电 | 降低助力，保留可骑行传动状态 |

## 6. 验证计划

| 测试 | 目的 | 通过标准方向 |
| --- | --- | --- |
| 巡航测试 | 验证目标 rpm 稳定性 | 平路和轻微起伏下踏频稳定，系统不过度调整 |
| 爬坡脚感一致性测试 | 验证坡度和踩踏力矩响应 | 陡坡中踩踏力矩落在舒适中间区，不以固定踏频为唯一目标 |
| 急加速测试 | 验证运动意图识别 | 快速加力后允许踏频上扬，电机先响应，齿比平滑增加 |
| 冲刺后高速巡航恢复测试 | 验证从冲刺回到巡航的策略 | 力矩下降后目标踏频回到巡航值，车辆以更高齿比保持高速巡航 |
| 起步测试 | 验证舒适起步比 | 起步不重、不空、不突兀 |
| 滑行恢复测试 | 验证预选传动比 | 恢复踩踏时阻力匹配当前车速 |
| 负载换比测试 | 验证高扭矩连续变比 | 高助力、高扭矩下无明显顿挫或不必要保护 |
| 曲柄相位测试 | 验证最优换比窗口 | 变比时脚感冲击低于未优化策略 |
| 控制优先级 A/B 测试 | 对比恒踏频优先和脚感力矩优先 | 陡坡场景中脚感力矩优先策略的自然度和疲劳感更优 |
| 误判恢复测试 | 验证用户可控性 | 用户通过 rpm、模式或临时偏好快速纠正系统 |
| 安全降级测试 | 验证故障边界 | 关键传感器异常时进入可骑行安全模式 |
| 体感评估 | 验证产品价值 | 自然、平顺、响应、噪音、控制感达到目标评分 |

## 7. 当前不做的承诺

以下内容可以作为研究方向，但不应在当前阶段写成确定性量产承诺：

- 具体峰值功率、持续功率、峰值扭矩和持续扭矩。
- 最终重量。
- 45 dB 绝对噪音承诺。
- 300 N·m 负载换挡承诺。
- 520% 变比范围承诺。
- 免维护终身承诺。
- 能量回收续航提升比例。
- “AI”作为外部主卖点。

这些指标需要后续台架、样车、热管理、效率、寿命和法规验证后再进入规格表。

## 8. 后续研究重点

优先级最高的后续研究不是“是否有自动换挡”，而是：

> 系统如何像汽车自动变速箱一样，根据骑手输入和路况判断巡航、冲刺、爬坡、滑行恢复、起步等意图，并在“踏频目标、踩踏力矩舒适区、传动比、电机转矩”之间动态切换控制优先级。

建议下一步输出三份更细文档：

1. `e-CVT MGU 意图识别控制逻辑草案`：定义状态机、传感器输入、触发条件和控制输出。
2. `e-CVT MGU 骑手 HMI 与 App 设置草案`：定义骑行中界面和 App/设置页的参数层级。
3. `e-CVT MGU 台架与样车验证矩阵`：把巡航、爬坡、冲刺、起步、滑行恢复、负载换比和安全降级变成可测任务。
