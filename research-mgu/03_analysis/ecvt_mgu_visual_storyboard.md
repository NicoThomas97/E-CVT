# e-CVT MGU 图文介绍网页视觉故事板

最后更新：2026-07-16  
定位阶段：图文网页 / 可视化介绍规划  
用途：指导后续 AI 生成图文介绍网页、曲线图、图解和动效，不作为控制算法或实测数据规范。

## 1. 可视化主线

网页要讲清楚的核心不是“这是一套自动变速电机”，而是：

> 这是一套以踏频管理为基础、以脚感一致性为目标的智能 e-CVT MGU 系统。它会根据骑手输入和路况，在巡航、冲刺、爬坡等场景中动态改变目标踏频、传动比和电机转矩策略。

视觉表达应从“恒踏频”升级到“脚感一致性控制”：

- 平路巡航：目标踏频优先，系统保持舒适 rpm。
- 平路突然加速：骑手意图优先，允许真实踏频短时上升，电机快速响应，齿比平滑增加。
- 冲刺结束：骑手踩踏力矩下降，目标踏频回到巡航值；由于齿比已经升高，车辆进入更高车速下的舒适巡航。
- 陡坡爬坡：脚下踩踏力矩优先，系统不死守固定踏频，而是动态改变目标踏频和齿比，让骑手踩踏力矩保持在不过重、不过空的舒适中间区。
- 换比窗口：e-CVT 变比优先放在踩踏力矩低谷 / 曲柄 12 点附近，减少脚感冲击。

## 3. 视觉编码规则

颜色和图形语义要稳定，避免每张图重新发明一套视觉语言。

| 类别 | 表示对象 | 推荐颜色 / 样式 | 说明 |
| --- | --- | --- | --- |
| 骑手输入 | 踩踏力矩、力矩变化率 | 深橙 / 黑橙实线 | 表达骑手主动输入和脚下负载 |
| 路况输入 | 坡度、车速 | 灰蓝 / 绿色实线 | 坡度可用阶跃线，车速可用平滑线 |
| 控制目标 | 目标踏频、力矩舒适区 | 虚线 / 半透明带 | 表示系统希望达到的区间，不是实测线 |
| 实际响应 | 真实踏频、真实车速 | 深蓝实线 | 与目标线区分开 |
| 控制输出 | 电机转矩、电流指令、e-CVT 传动比 | 蓝色 / 紫色实线 | 电机输出和传动输出要分清 |
| 意图判断 | 巡航、冲刺、爬坡、恢复巡航 | 顶部状态条 / 标签 | 用短标签标明系统当前判断 |
| 执行窗口 | 曲柄 12 点 / 力矩低谷 | 竖向淡黄色高亮带 | 标出建议变比时机 |
| 禁止承诺 | 未验证指标 | 灰色小注释 | 45 dB、300 N·m、520% 等只能写参考或研究目标 |

## 4. 核心曲线图规范

通用规则：

- 所有曲线图横坐标统一为时间。
- 不把所有参数塞进一个坐标轴，采用纵向堆叠小图。
- 每个小图只表达一个参数族，例如坡度、踏频、传动比、电机转矩、踩踏力矩。
- 曲线强调趋势，不代表实测数据；除非后续有台架或实车数据，否则不要标成精确数据图。
- 每张图必须标注“输入 / 判断 / 输出 / 体感结果”。
- 目标踏频和真实踏频必须分开表示，不能混成同一条线。

### 4.1 平路巡航突然加速曲线

场景设定：

- 坡度：约 3 度，保持低坡或缓坡。
- 初始状态：舒适巡航，目标踏频约 50 rpm。
- 骑手动作：突然加大踩踏力矩，表达加速 / 冲刺意图。
- 系统策略：电机转矩快速响应，e-CVT 齿比平滑增加，不立即强行压低踏频。
- 结束状态：骑手力矩下降，目标踏频回到约 50 rpm；齿比保持较高，车辆进入更高车速下的舒适巡航。

建议纵向小图，从上到下：

1. 坡度：约 3 度水平线。
2. 意图状态条：巡航 -> 冲刺 / 加速 -> 高速巡航。
3. 目标踏频：50 rpm；冲刺阶段可轻微上移或保持为“允许上扬区间”；冲刺结束回到 50 rpm。
4. 真实踏频：从 50 rpm 短时上升到 70-90 rpm 区间，随后回落到 50 rpm 附近。
5. e-CVT 传动比：比电机响应更慢、更平滑地上升；冲刺结束后保持在较高平台。
6. 电机转矩 / 相电流指令：在骑手力矩上升后快速上升，冲刺结束后回落到高速巡航所需水平。
7. 车速：由低速巡航逐渐升高，冲刺结束后保持更高车速。
8. 踩踏力矩：冲刺开始时幅值明显上升，冲刺结束后回落到中低水平。
9. 曲柄相位窗口：在力矩低谷 / 12 点附近标出可执行变比窗口。

关键注释文案：

- “冲刺时不急着压回固定踏频，先尊重骑手加速意图。”
- “电机先接住加速请求，e-CVT 再平滑升齿比。”
- “冲刺结束后目标踏频回到 50 rpm，但高齿比让车辆进入高速巡航。”

禁止误画：

- 不要把齿比画成瞬间跳变。
- 不要把真实踏频一直压在 50 rpm。
- 不要把冲刺结束后的车速画回原来的低速。

### 4.2 陡坡爬坡脚感一致性曲线

场景设定：

- 坡度：约 10 度，保持陡坡或进入陡坡。
- 初始状态：爬坡中，骑手持续输出。
- 骑手需求：不是单纯固定踏频，而是希望脚下踩踏力矩处于舒适中间值。
- 系统策略：以踩踏力矩舒适区为主目标，动态调整目标踏频和 e-CVT 齿比。
- 体感目标：既不沉重踩不动，也不空、不虚。

建议纵向小图，从上到下：

1. 坡度：约 10 度高平台。
2. 意图状态条：爬坡 / 陡坡脚感控制。
3. 踩踏力矩舒适区：用半透明横向带表示中间力矩区。
4. 踩踏力矩：围绕舒适区波动；当力矩偏大，系统减轻传动负担；当力矩偏小，系统避免过度变轻。
5. 目标踏频：不是固定水平线，而是随脚感控制动态变化。
6. 真实踏频：跟随目标踏频变化，但不作为唯一控制核心。
7. e-CVT 传动比：根据踩踏力矩偏差动态调整，目标是把脚下力矩拉回舒适区。
8. 电机转矩 / 相电流指令：提供支撑，但不把脚感变得过空。
9. 车速：可相对稳定或缓慢变化，不作为第一目标。

关键注释文案：

- “陡坡时不死守固定踏频，脚下力矩一致性优先。”
- “目标踏频在这里是调节脚感的工具，而不是绝对固定目标。”
- “系统通过齿比和电机转矩协同，把踩踏力矩拉回舒适中间区。”

禁止误画：

- 不要把爬坡曲线画成单纯恒踏频控制。
- 不要让踩踏力矩一路升高到很重还不处理。
- 不要把电机转矩画成越大越好；重点是支撑感和脚感一致。

### 4.3 曲柄相位换比窗口图

场景设定：

- 踩踏力矩呈周期波形。
- 每个周期在曲柄接近 12 点附近出现力矩凹陷。
- e-CVT 变比优先选择在这个低力矩窗口执行。

建议图形：

- 左侧：曲柄圆周小图，标出 12 点位置。
- 右侧：一段踩踏力矩波形，标出低谷。
- 在低谷处放竖向高亮带，标注“推荐变比窗口”。
- 用对比小图展示：高力矩区变比 = 脚感冲击大；低力矩区变比 = 脚感更自然。

关键注释文案：

- “变比时机不只看时间，也看脚踩力矩波形。”
- “在力矩低谷执行变比，可降低脚下突兀感。”
- “曲柄相位是 e-CVT 脚感控制的重要输入。”

## 5. 图解框图规范

### 5.1 多传感器意图识别框图

结构：

`传感器输入 -> 意图识别 -> 控制目标选择 -> 控制输出 -> 骑行体感`

输入项：

- 踩踏力矩。
- 踏频。
- 曲柄位置。
- 车速。
- 坡度 / IMU。
- 刹车状态。
- 电机电流 / 转速。
- 电池与温度状态。

意图状态：

- 舒适巡航。
- 运动巡航。
- 突然加速 / 冲刺。
- 陡坡爬坡。
- 滑行恢复。
- 停车起步。

输出项：

- 目标踏频。
- 目标踩踏力矩区间。
- e-CVT 目标传动比。
- 电机转矩 / 电流指令。
- 再生制动限制。
- HMI 状态提示。

### 5.2 电机转矩与 e-CVT 协同图

核心表达：

- 电机转矩负责快速响应骑手意图。
- e-CVT 传动比负责中慢速整理节奏和脚感。
- 两者共同决定真实踏频、车速和脚下力矩。

推荐文案：

> 电机先接住意图，e-CVT 再整理节奏。冲刺时允许踏频上扬，爬坡时优先稳住脚下力矩。

## 7. 文案口径

可用标题：

- 不只是自动变速，而是脚感管理
- 巡航看踏频，冲刺看意图，爬坡看脚感
- 电机先接住意图，e-CVT 再整理节奏
- 陡坡时，固定踏频不是第一优先级
- 在力矩低谷变比，让脚感更自然

可用短句：

- “目标踏频不是铁律，它会为脚感服务。”
- “冲刺时允许踏频上扬，爬坡时稳住脚下力矩。”
- “系统不是替骑手做决定，而是读懂骑手已经表达出的意图。”
- “曲柄相位让变比动作发生在最不打扰骑手的瞬间。”

避免使用：

- “全自动永远保持固定踏频。”
- “任何坡度都无感。”
- “已实现 300 N·m 负载换挡。”
- “已达到 45 dB 噪音。”
- “量产 520% 变比范围。”

## 8. 验收检查

后续用本故事板生成网页草图时，至少检查：

- 是否正确区分巡航、冲刺、爬坡三种控制优先级。
- 是否表达“冲刺允许踏频上升，冲刺后高齿比巡航”。
- 是否表达“陡坡爬坡以踩踏力矩舒适区优先”，而不是误画成固定踏频优先。
- 是否明确显示曲柄 12 点 / 力矩低谷换比窗口。
- 是否把目标踏频和真实踏频分开。
- 是否没有把研究目标或竞品参数写成量产承诺。
