# e-CVT MGU Go / No-Go 标准草案

最后更新：2026-07-05

## 1. 草案定位

这不是最终立项结论，而是第一阶段证据补齐后的判断框架。它用于决定哪些场景值得进入下一阶段台架、样机和 OEM 访谈。

核心问题是：

> 在某个整车场景中，e-CVT MGU 是否比电子外变、花鼓内变/CVT、有级 gearbox MGU、普通中置电机带来明显更高的 OEM 系统收益？

## 2. 场景级 Go 条件

某个场景进入 Go 候选，需要同时满足：

- 用户痛点强：频繁停启、坡道、载重、泥沙冲击、低维护或不会换挡是真实痛点。
- 替代方案不完整：电子外变、花鼓 CVT/内变、有级 MGU 或普通中置电机无法以更低复杂度解决主要痛点。
- OEM 有架构收益：后轮简化、车架布置、皮带友好、诊断、安全、售后或品牌差异化可被整车厂感知。
- 用户价值能被表达：不是“技术更复杂”，而是更省心、更平顺、更少维护、更不容易坏。
- 成本和重量有承载空间：整车价格带足够高，用户和 OEM 愿为系统收益付费。

## 3. 场景级 No-Go 条件

某个场景应暂时 No-Go，如果出现任一情况：

- hub CVT/内变已经以更低成本和复杂度解决 80% 以上核心痛点。
- 电子外变已经满足目标用户对自动/运动控制的主要需求，且外露传动不是高强度痛点。
- 用户主要购买理由是轻量、低阻、便宜或传统运动手感。
- 整车 OEM 无法接受单一高复杂度中置系统带来的售后和供应风险。
- e-CVT MGU 的价值只能用参数解释，不能转化成整车体验或 TCO。

## 4. 技术门槛

以下指标需要进入下一阶段台架定义，当前不应拍脑袋给数字：

| 维度 | Go 需要证明 | No-Go 风险 |
| --- | --- | --- |
| 效率 | 常用踏频/扭矩/坡度区间效率可接受，且不显著吞掉续航价值 | 无级机构在高载低速或高速巡航效率损失过大 |
| 热 | 长坡、满载、低速和高温环境下不出现不可接受热衰减 | 峰值功率好看但持续输出不可用 |
| 寿命 | 连续变速机构、轴承、密封、润滑和执行器寿命可支撑目标车型 | 传动寿命或密封成为核心售后风险 |
| 控制 | 目标踏频、虚拟档位、手动覆盖、站姿重踩和湿滑路面控制可预测 | 骑手觉得系统抢控制或反应不可预期 |
| 扭矩响应 | 参考 Avinox Tech Details，证明 fast response with no overshoot、assist remain/overrun 和 constant torque output 可形成自然骑感 | 响应过冲、拖拽、overrun 过长或用户觉得踏感怪 |
| 负载换挡 | 参考 Avinox Tech Details，明确 300 Nm、<0.1 s 等目标的测量位置和可重复工况 | 图卡值无法转化为可测指标，或高载换挡造成冲击/噪声/寿命风险 |
| NVH | 电机、变速执行和链/带路径噪声符合高端整车预期 | 高频啸叫、档位模拟噪声或链路噪声破坏高级感 |
| 包装重量 | 对目标车型的重量、Q-factor、离地间隙和车架结构影响可接受 | 比替代方案重太多或限制车架设计 |
| 断电降级 | 低电、故障或断电时有清晰可骑/可推/可维修策略 | 系统失效导致车辆难以使用或维修 |
| 售后诊断 | 故障码、远程诊断、模块更换和固件升级流程可给 OEM | 售后只能整体换大总成，成本失控 |
| 系统功能 | mechanical lock、reverse assist、built-in speed sensor、charging 等功能有真实装车价值和安全边界 | 功能只适合展示，量产后难以解释或引入法规/失效风险 |
| 供应链 | 关键部件可控，量产一致性和品质闭环明确 | 变速核心件、传感器或控制器供应风险过高 |

## 5. 场景价值假设分组

本节不再使用单一“首发排序”。当前更重要的是识别不同场景中 e-CVT MGU 的价值来源、替代压力和验证门槛。

| 分组 | 场景 | 当前判断 |
| --- | --- | --- |
| 强痛点 / 高验证门槛 | High-end eMTB / gravity / technical climbing | 后拨/飞轮/链条外露、簧下质量、技术爬坡和负载换挡痛点强；Avinox Tech Details 已把 >1.1 kg、扭矩平滑、<0.1 s 和 300 Nm 作为量产方向/准量产目标，但必须证明重量、效率、NVH、冲击耐久和骑手信任。 |
| 强痛点 / 强替代压力 | Cargo / family utility | 满载起步、坡道、频繁停启、低技能用户和低维护痛点强；Avinox 的 reverse assist、mechanical lock、stationary/load shift 提高了本场景重要性，但 hub CVT/内变、有级 MGU 是强替代，必须证明高载热、寿命、TCO 和后轮维护收益。 |
| 综合价值 / 中高替代压力 | 高端 SUV / trekking / all-road commuter | 低维护、目标踏频、轻越野、长距离、载物、built-in speed sensor 和 premium integrated drive 价值可以组合；但 hub CVT/内变和有级 MGU 已能覆盖一部分需求。 |
| UX 价值 / 高替代压力 | Premium urban commuter | 用户不用操心换挡和 connected/security 有价值，但 enviolo、Owuru、普通内变或轮毂路线已经能解决很多城市需求。 |
| 可靠性导向 / 高替代压力 | Touring / adventure | 低维护和长途可靠性需求真实，但 Rohloff、Pinion gearbox、有级 MGU 的可靠性叙事强。 |
| 价值弱 / 重量效率敏感 | Lightweight eMTB / eGravel | 轻、自然、低阻和运动控制优先，e-CVT MGU 的重量、体积和效率代价很难被低维护收益覆盖。 |

## 6. 下一阶段证据任务

### 6.1 台架任务

- 建立 e-CVT 效率 map，与电子外变、花鼓 CVT/内变、有级 MGU、普通中置传动对照。
- 建立满载起步、长坡、低速高扭矩、频繁停启热循环。
- 定义目标踏频保持质量：响应速度、超调、抖动、用户可感知延迟。
- 定义 Avinox Tech Details 对标测试：扭矩平滑、<0.1 s 换挡窗口、300 Nm 负载换挡、45 dB NVH、3x durability、10% efficiency claim。
- 定义寿命和污染工况：泥水、粉尘、冲击、温度、低电压。

### 6.2 整车任务

- SUV/trekking、cargo/family、eMTB 各做一套整车架构包络。
- 对比后轮结构、链/带路径、悬架/货架/车架空间收益。
- 评估 service kit：整机换件、油液/密封、诊断端口、软件升级。

### 6.3 OEM 访谈任务

每个目标 OEM 至少问清：

- 他们当前对电子外变、hub CVT/内变、有级 MGU 的最大不满是什么。
- 他们愿意为哪些系统收益提高 BOM。
- 他们最担心的售后和供应链风险是什么。
- 哪个细分车型最能承载高价一体化系统。

## 7. 当前草案判断

进入下一阶段最合理的不是“全面做 e-CVT MGU”，而是把价值假设拆成可验证问题：

- eMTB：验证后拨取消、后轮/悬架自由度、技术爬坡和骑手控制是否能形成可感知收益。
- Cargo / family utility：验证满载停启、低技能用户、低维护和 TCO 价值。
- 高端 SUV / trekking：验证 premium integrated low-maintenance、目标踏频和后轮简化价值。

Avinox Tech Details 已经给出了足够强的量产方向/准量产目标，本项目不能再把 Avinox 当作远期概念展示低估；但在没有重量、效率、NVH、热、耐久、故障降级、charging 边界和售后证据前，不应把任何一个场景写成最终立项结论。
