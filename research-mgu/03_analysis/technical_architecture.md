# 中置变速电机系统架构级技术分析

调研日期：2026-07-03  
深度：系统架构级，不进入齿形、轴承、润滑、控制器原理图等详细设计。

## 1. 系统边界

目标产品不是普通中置电机，而是“中置驱动 + 变速/传动控制”系统。可选系统边界有三种：

1. `真正 MGU`：电机、减速、骑行变速机构、换挡执行、控制器高度集成。Pinion E-Drive MGU 是已量产参照（O001），Gobao X-system 是官方 eCVT MGU 参照，Avinox MG Concept 是量产方向/准量产产品定义参照（V032/F043-F048）。
2. `准 MGU`：中置电机仍无内置变速，依靠电子后拨、内变花鼓、CVT 花鼓或电子 gearbox 做系统级自动换挡。Bosch eShift、Shimano AUTO/FREE SHIFT、SRAM Eagle Powertrain 属于这一类（O004, O006, O008）。
3. `替代传动生态`：不做电机集成，但通过 gearbox、内变、CVT hub、belt drive 或高耐久外变解决传动痛点。它们不是同类竞品，但会影响用户是否愿意为 MGU 付费。

## 2. 关键架构分叉

### A. 有级 gearbox vs eCVT/无级

- 有级 gearbox：更容易定义机械速比、效率和失效模式；Pinion MGU 的 9/12 档、568%/600% 速比是清晰参照（O001）。
- eCVT/无级：用户体验上更接近“无感自动变速”，但效率、热、控制稳定性、寿命验证和成本风险更高；Gobao X-system 已有官方规格块，Avinox MG Concept 已有 Tech Details 量产方向/准量产图卡，可作为功能目标和验证目标（V004, V032）。
- 外部 CVT hub：enviolo/Owuru 类路线能提供自动/无级体验，但重量、扭矩、效率、后轮簧下质量和整车布置要单独评估（O009, O022）。

### B. 变速机构放在哪里

| 位置 | 优点 | 风险 |
| --- | --- | --- |
| 电机壳体内 / bottom-bracket MGU | 车尾更干净、传动寿命可能更好、适合 belt drive、系统控制闭环强 | 结构复杂、散热/润滑/密封/NVH/维修难度集中 |
| 后花鼓内变 / CVT hub | 可复用成熟车架中置电机，自动换挡体验可快上市 | 后轮重量、轮组维护、效率和高扭矩寿命需验证 |
| 电子后拨/外变 | 成本与服务网络更友好，运动用户熟悉 | 链条/飞轮磨损仍在，低维护卖点弱，外露件抗冲击差 |

### C. 控制系统

MGU 需要同时控制助力输出和传动比。最低可用传感/控制闭环应考虑：

- 骑手输入：踏频、扭矩、踩踏相位、曲柄转速。
- 整车状态：车速、坡度估计、加速度、档位/传动比、温度、电池状态。
- 换挡状态：目标比、实际比、执行器位置、电机卸载/补偿策略、失败回退。
- 用户模式：手动、自动、爬坡、节能、货运、牵引/推行、维护诊断。

Avinox Tech Details 把控制目标进一步具体化：扭矩平滑、fast response with no overshoot、assist remain/overrun、<0.1 s shifting window、constant torque output、automatic/manual/multi-gear shifting、mechanical motor lock、electric reverse assist、built-in speed sensor 和 downhill charging 图示（V032/F044-F047）。这些应作为量产方向/准量产级控制与系统功能输入。

Bosch/Shimano/SRAM 的价值不只是硬件，而是“电机控制 + 换挡行为 + 显示/APP + OEM 适配”的系统化能力（O004, O006, O008）。自研 MGU 如果只完成机械集成，没有做好换挡策略和整车体验，很难形成用户可感知优势。

## 3. 研发风险清单

### 机械与热

- 电机峰值功率、减速器、骑行变速机构和换挡执行器集中在中置壳体，会放大热管理和润滑设计难度。
- eMTB 冲击、水泥沙、清洗、摔车和高负载爬坡会同时考验密封、轴承、齿轮、传感器和线束。
- 量产 MGU 的重量大概率难以接近轻量中置电机；应把重量目标与目标车型绑定，不要用 eMTB race 指标约束 cargo/SUV 方案。

### 换挡体验

- 自动换挡不能只追求“能自动”，必须处理高踏频、低速大扭矩、技术爬坡、跳跃落地、急加速、空踩和停启。
- 手动模式仍然必要，尤其是 eMTB 用户会要求可预测、可锁定、可快速干预。
- 变速比范围需要覆盖爬坡和高速巡航；Pinion 568%/600% 是量产参照，不代表自研必须照搬，但可作为上限锚点（O001）。
- Avinox 的 100%-520% 连续变比和 1-13 虚拟档位应作为 eCVT MGU 的量产方向/准量产目标锚点；自研方案如果显著低于该范围，需要有明确场景理由。

### 生态与服务

- MGU 一体化会把维修边界从“换链条/飞轮/后拨”转移到“整机返修/授权维修”。
- OEM 会关心 mounting interface、Q-factor、链线/皮带线、车架空间、电池/显示/APP、诊断工具、售后备件。
- Gates belt、内变、gearbox 生态说明低维护传动卖点成立，但也意味着车架接口和供应链约束更强（O013, O024）。

## 4. 推荐的技术路线假设

第一轮不要直接定“eCVT”或“有级 gearbox”，建议并行验证两条路线：

1. `有级 MGU MVP`：9-12 档或等效宽速比，优先验证效率、换挡负载、密封、NVH、服务成本；对标 Pinion MGU。
2. `自动控制外部化 MVP`：中置电机 + 电子变速/内变/CVT hub，快速验证用户对自动换挡的真实价值；对标 Bosch/Shimano/SRAM/enviolo。

如果外部化 MVP 的用户价值足够强但低维护价值不足，再进入真正 MGU；如果用户明确为低维护和整车布置付费，再扩大机械一体化投入。

## 5. 下一步实验

- 台架：效率地图、热稳态、峰值/持续扭矩、换挡冲击、执行器寿命、密封进水、泥沙污染。
- 整车：eMTB 技术爬坡、货运满载起步、城市停启、长坡热衰减、清洗后功能。
- 软件：自动换挡策略 A/B、手动覆盖、故障降级、APP/显示交互、诊断记录。
- 商务/服务：整机维修时间、备件成本、OEM 安装工时、车架接口变更成本。

## 6. 关键证据缺口

- Avinox MG Concept 内部结构图、重量、Q-factor、效率/热/寿命测试基准、charging 传动路径、断电降级和正式量产 datasheet 缺口；Tech Details 图卡值已可作为量产方向/准量产目标输入。
- Gobao X1/X1P 官方详情页、样机图、量产节奏和 OEM 客户缺口。
- Valeo/Revonte/Owuru 当前商业状态缺口。
- SRAM/enviolo/Rohloff/3X3 等准直接或替代路线的最新官方 compatibility matrix 缺口。
- L3 高性能中置电机的当前型号参数需要二轮逐项抽取，避免混用年份、地区和固件口径。
