# 中置变速电机研发预研报告 v5.1

调研日期：2026-07-04  
定位：修正“收益最大化”的定义，转向 OEM 架构选择问题。

## 1. 问题重定义

这里的收益最大化不是我们作为供应商能赚多少钱，而是整车公司为什么会选择 MGU 架构。

真正问题是：

> 如果 OEM 可以选择中置电机 + SRAM/Shimano 电子外变、中置电机 + Rohloff/enviolo/3X3 花鼓内变/CVT、普通高性能中置电机，为什么还要选择中置电机集成变速？

MGU 只有在它给 OEM 带来的系统收益超过重量、成本、售后复杂度、供应商锁定和开发风险时，才是好选择。

## 2. 当前初步答案

MGU 最可能成立的场景不是“所有 e-bike”，而是：

1. `高端 SUV / trekking`
2. `cargo / family utility`
3. `高端 touring / adventure`

这些场景共同点是：用户对低维护、可靠性、满载/停启、长续航、宽速比和省心体验更敏感，对重量和成本相对不如轻量运动用户敏感。

## 3. 为什么不是直接押 eMTB

高端 eMTB 有战略声量，但不是最稳首发场景。

原因：

- SRAM/Shimano 电子外变已经较好地解决运动用户的换挡体验。
- eMTB 用户对重量、NVH、手动控制、冲击耐久和可预测性更敏感。
- MGU 必须证明后端简化、技术爬坡和低维护优势足够大，才能抵消重量和复杂度。

因此 eMTB 更适合作为技术展示和上限验证，而不是唯一首发市场。

## 4. 为什么 SUV / trekking / cargo 更强

这些场景里，MGU 的系统收益更集中：

- 相比电子外变：MGU 能减少外露传动维护和损坏。
- 相比花鼓内变/CVT：MGU 能把复杂度集中在中置单元，减少后轮复杂度，并更容易做整车诊断。
- 相比普通中置电机：MGU 能提供更强的低维护和高端差异化叙事。

Simplon KAGU :ePinion High、Pinion e-trekking 装车页和 Radfahren/ElektroRad 测试都支持这个方向更自然（V019, V020, V022）。

## 5. 后续如何使用 pm-skills

`pm-skills` 后面会用，但不能替代证据研究。

使用顺序应该是：

1. 先完成 `scenario_substitutability_matrix.csv` 和商业/用户证据。
2. 再用 `pm-skills competitive-analysis` 整理 MGU、电子外变、花鼓内变/CVT、普通中置电机的定位。
3. 只对排名靠前的场景使用 `pm-skills market-sizing`。
4. 当首发场景锁定后，再用 PRD/产品策略类 skill 组织产品定义。

不应该让 `pm-skills` 直接判断哪个场景成立，也不应该让它发明市场规模或用户付费意愿。

## 6. 下一步

下一步应围绕 OEM 架构选择继续补证：

- 每个场景下，电子外变、花鼓内变/CVT、普通中置电机分别解决了哪些痛点？
- MGU 的增量收益是否足够大？
- OEM 是否能用这个增量收益支撑更高售价、更强差异化或更低售后风险？
- 哪些场景只是“MGU 可以用”，哪些场景才是“MGU 明显更值得用”？
